方案 A · Listing 视频流里浮 Community Chip

**目标 buyer**:被动派发型 — 想刷视频,不想做选区决策。Listing 视频是主轴,community 是上下文。

  • 左下角:listing 元数据(房价、地址、户型、agent)— 卖房买房主线永不消失
  • 左上角浮一个 community chip(脉冲动画引导第一次注意,不挡 listing 元数据)
  • 点击 chip → 底部 sheet 弹起(L1)— sheet 只装信息和缩略图,**不在 sheet 内播视频**
  • Sheet 内点 community 视频缩略图 / 房源 / "Browse all" → 跳转**全屏新页 L2**,顶部明示返回路径 ← 14207 NE 23rd Pl
  • 顶部 HUD 实时显示 L0 / L1 / L2,验证不会迷路
L0 listing feed → L1 sheet → L2 全屏页 每层都有明确返回路径 没有 modal 内播全屏视频

试一下: 点左上 chip → 进 sheet → 点任意 community 视频缩略图 / 房源缩略图 → 看顶部 ← 14207 NE 23rd Pl 把你带回原 listing。

Scenario B · Search → Communities → Videos → Homes

Buyer 类型:主动选区 — 还没锚定 listing,先比较 community,再看里面的房。

  1. L0 · Community Grid:Search → segmented "Communities" → 412 个小区卡片
  2. L1 · Community Feed:点卡片进 community 视频 feed(上下滑 切 5 个视频),底部挂 🏠 47 homes here chip
  3. L2 · Listings Sheet:点 chip → bottom sheet 列出 47 个 listing
  4. L3 · Listing Carousel:点 listing → 全屏 listing 详情(左右滑 切 listings)

Back 路径:每层都回上一层(L3 → L2 → L1 → L0),不跳层。

关键差异 vs A:Listings chip 只在 B 里出现 —— 因为 B 用户没有锚定 listing,跨链是自然的转化路径;A 用户已经锚 listing,跨链反而稀释意图。

L1 上下滑 · L3 左右滑 Listings chip 只在 B 闭环回 community feed

Create Community 三步化

  • Step 1: Name + 地图画范围 (or ZIP codes) — 必填
  • Step 2: 1 句描述 + 1 张封面图 — 必填
  • Step 3: [Skip] 或 高级设置(学区/walk score/tags 等)— 全部选填,默认值兜底,创建后随时改

设计原则:用户还没体验过 community 是什么时,不让他在表单里炸晕。其余 17 个字段全部推到 community 详情页"⚙️ Settings"里渐进披露。
